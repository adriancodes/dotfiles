NAME
gcloud preview docker - ==SUPPRESS==
SYNOPSIS
gcloud preview docker   [EXTRA_ARGS   …]   [--authorize_only,   -a]   [--docker-host   DOCKER_HOST]   [--format   FORMAT]   [--help]   [--project   PROJECT]   [--quiet,   -q]   [--server   SERVER,   -s   SERVER;   default="https://container.cloud.google.com"]   [-h]
DESCRIPTION
The docker sub-command of gcloud wraps docker commands, so that gcloud can inject the appropriate fresh authentication token into requests that interact with the docker registry. As commands are simply passed through to docker, see this for a full reference of command-line options that can be supplied after the —  http://docs.docker.com/reference/commandline/cli/
POSITIONAL ARGUMENTS
[EXTRA_ARGS …]
Arguments to pass to docker
FLAGS
--authorize_only, -a
Configure docker authorization only, do not launch the docker command-line
--docker-host DOCKER_HOST
The URL to connect to Docker Daemon. Format: tcp://host:port or unix:///path/to/socket.
--server SERVER, -s SERVER; default="https://container.cloud.google.com"
The address of the Google Cloud Registry
GLOBAL FLAGS
--format FORMAT
Specify a format for printed output. By default, a command-specific human-friendly output format is used. Setting this flag to one of the available options will serialize the result of the command in the chosen format and print it to stdout. Supported formats are: json, text, yaml.
--help
Display detailed help.
--project PROJECT
The Google Cloud Platform project name to use for this invocation. If omitted then the current project is assumed.
--quiet, -q
Disable all interactive prompts when running gcloud commands. If input is required, defaults will be used, or an error will be raised.
-h
Print a summary help and exit.
EXAMPLES
Pull the image container.cloud.google.com/my-org/my-image from the docker registry: gcloud preview docker — pull container.cloud.google.com/my-org/my-image
Push the image container.cloud.google.com/foo/bar:baz to our private docker registry. gcloud preview docker — push container.cloud.google.com/foo/bar:baz
Configure authentication, then simply use docker: gcloud preview docker --authorize_only docker push container.cloud.google.com/foo/bar:baz
NOTES
This command is in the Google Cloud SDK preview component. See installing components if it is not installed.
